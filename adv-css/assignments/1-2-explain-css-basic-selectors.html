1-2: Explain CSS Basic Selectors

Element Selector

An Element Selector is exactly what it's title as; it selects a specific HTML element across the entire page (if used imbedded), the element (if inline), or site (if style sheet.). The board element selector is best used when you're trying to achieve an universal style of some kind. Background colors are great examples. Whether it's for a universal nav, footer, or body. It is identified as;

element {

attribute: value;

}

Group Selector

The group selector is exactly like the Element Selector, except is selects multiple html elements. When the same CSS code needs to be applied to multiple elements, the group selector saves time rewriting the CSS and space in the code. It's best used for universal styles that need to be applied across multiple elements, like if you want the same background color for both the nav and footer to give the site a start and finish visual cue, almost like a sandwich. The code is identicle to the Element selector, except there is a , between the seperate elements.

Descendant Selector

Descendant selectors are almost like if-then type selectors. The CSS will only apply to an element if they meet the two or more requirements. This is best used when you use multiple elements of the same kind, but they are for different things. For example, if I want values to only appear in the top nav, but my site also has a side nav I could create a 'top-nav' div and write a css rule like:

top-nav nav {

property: value;

}

This way, the descendant selector will only select the nav if it is within the top-nav div

Class Selector; dependent & independent

The class selector selects an html element that has been assigned a specific class. This is best used when wanting a more percise selections but still having the option to target multiple pages. An exmaple for this would be if you have multiple articles in the body section. You can give a 'main' class to the most important piece in the body, which will allow you to easily target that with CSS that will make it stand out to the users that 'hey! this is important!' design. The difference between dependent & independent is that dependent must be attached to an element within the CSS. For example:

p.class {

property: value;

}

this dependent class selector will select all elements given the 'class' class, but only if they are within a paragraph element.

.class {

property: value;

}

this independent class selector will select all elements given the 'class' class regardless of other elements.

ID Selector; dependent & independent

The ID selector is just like the class selector except that IDs can only be used once, which makes them even more percise and unique. They are designated by a # in replace of the . of a class. The difference between dependent and independent remain the same.

Universal Selector

The universal select will apply to every element. This is best used when you need to apply a basic css rule to everything. For example, if you wanted the exact same font for every piece of type in you page, you could write the font style in the universal selector. But that wouldn't be the best design choice since it would get boring and too repetive for the user.

Pseudo Selector

A pseudo selector is the most unique selector, as it targets a state of an element rather than an element itself. The most common example is the mouse hover. When the mouse hovers over a button, it can target that state and turn it into a different color. This is written as:

selector:pseudo-class {

property: value;

}

Specificity hierarchy

They are in order from lowest to highest: Element, Group, Descendent, Class, ID
